# Folio — Release Notes

## Release 1.1

---

### New Features

**Toast Notifications**
Replaced all browser `alert()` dialogs with a non-blocking toast notification system. Toasts appear at the bottom-centre of the screen, animate in, auto-dismiss after 3 seconds, and stack if multiple fire at once. Covers audio, ebook, transcript, and cover file validation errors.

**Keyboard Shortcuts**
Shortcuts fire when the player is open and no text input is focused.

| Key | Action |
|-----|--------|
| `Space` | Play / pause |
| `←` | Nudge back one sentence |
| `→` | Nudge forward one sentence |
| `[` | Speed −0.1× |
| `]` | Speed +0.1× |
| `F` | Toggle fullscreen |

**Wake Lock**
The screen stays on during playback. Lock is acquired on play, released on pause/stop, and re-acquired if the tab returns to the foreground while playing. Fails silently on unsupported browsers.

**Media Session API**
Book title and artist ("Folio") appear in the OS media overlay and lock screen controls. Play, pause, and seek actions are wired to the player. Fails silently if unsupported.

**Page Title Updates**
`▶ Title — Folio` while playing, `Title — Folio` while paused, `Folio` on the library screen.

**Nudge Button Press Feedback**
Nudge buttons (−10, −5, −1, +1, +5, +10) scale down slightly on press for tactile feedback.

**Smooth Theme Switching**
Theme changes animate over 200 ms instead of snapping instantly.

---

### Performance

**Ebook Loading**
The ebook DOM is now built in async chunks, yielding to the browser every 16 ms. On long books this eliminates a 20–60 second UI freeze — text appears progressively and the reader is usable immediately.

**Transcript Sync**
Transcript timing matching now runs in an inline Web Worker so it never blocks the main thread. The reader is fully usable while timings build in the background. A chunked async fallback is used automatically in environments that block blob workers.

**EPUB Base64 Encoding**
Replaced slow character-by-character string loops with native browser encode/decode paths (`FileReader.readAsDataURL` and `fetch` data URLs), eliminating multi-second stalls when opening large EPUB files.

**TOC Building**
Table of contents construction is deferred via `requestIdleCallback` (with a `setTimeout` fallback) so it does not compete with the initial render frame.

**Cover Image Resizing**
Cover images are resized to a maximum of 400 × 600 px at 85% JPEG quality before being stored, preventing large photos from consuming storage quota.

**Debounced Progress Saving**
Progress writes are debounced at 500 ms so rapid sentence advances only trigger a single storage write.

---

### Storage

**IndexedDB for Large Content**
Large book data (ebook text, transcript JSON, cover images) is now stored in IndexedDB rather than localStorage. Only small metadata (title, progress, settings) remains in localStorage. This eliminates the `QuotaExceededError` that occurred when adding books with large content, particularly when running from `file:///` where all local files share the same storage origin. Deleting a book cleans up both storage locations.

---

### Bug Fixes

**Seek Bar & Skip Buttons Did Not Sync Ebook**
Dragging the seek bar or tapping the skip buttons while paused now immediately moves the ebook highlight to the matching sentence. Previously the ebook only updated while audio was actively playing.

**TTS Play Button State**
The play button now correctly shows the playing/paused state in TTS mode. Previously it could get stuck showing ▶ after pausing because the state check used an internal flag that was not being reset correctly.

**TTS Skip Buttons Now Work**
The ⏮ / ⏭ skip buttons were no-ops in TTS mode and left the speech engine in a broken state. They now jump forward or back by 3 sentences and restart speech from the new position.

**TTS Sentence Navigation**
Tapping any sentence in the ebook while TTS is playing or paused now immediately jumps to that sentence. Previously clicking only moved the highlight but the speech engine continued from wherever it was.

**TTS Word Highlighting**
Word-by-word highlighting now works in TTS mode. The browser's `SpeechSynthesisUtterance.onboundary` event is used to track the current word as speech progresses. Previously word highlighting only worked in audio+transcript mode.

**Position Not Saved on Library Navigation**
Navigating back to the library now performs an immediate synchronous save of the current position before resetting any state. Previously the debounced save could be swallowed if the user navigated away quickly.

**Position Not Restored on Reopen**
Returning to a book from the library, or refreshing the page, now correctly restores both the audio cursor and the ebook scroll position. The page title and progress indicator update immediately on open.

**Refresh Returned to Library**
Refreshing the page while reading now reopens the book directly instead of returning to the library screen.

**Ebook Resync After Transcript Loads**
When a book with a transcript is opened, the ebook now silently corrects to the transcript-matched position once timing data finishes building in the background. It also snaps to the correct sentence the instant play is pressed, before the first `timeupdate` event fires.
