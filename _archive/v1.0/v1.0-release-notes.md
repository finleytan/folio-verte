# Folio Reader — Release Notes

## v1.0 — Initial Release

---

### Overview

Folio is a single-file HTML progressive web app for reading and listening to books. The entire application — UI, logic, parsers, and styles — lives in one self-contained HTML file with two companion files (`manifest.json` and `sw.js`) for PWA installation and offline support.

The initial release delivers a fully functional audiobook and ebook reader with word-level transcript synchronisation, text-to-speech, and a file-system-based library for desktop and mobile use.

### Core Architecture

**Single-file PWA**
The entire application ships as one HTML file. There is no build step, no bundler, and no server required. The file can be opened directly in a browser from a local folder, served from any static host, or installed as a PWA on mobile and desktop devices.

**Storage model**
- Library metadata and reading progress are stored in `localStorage` under versioned keys (`folio_library_v2`, `folio_pwa_progress_v1`, `folio_display_prefs_v1`)
- Large ebook content, transcript data, and cover images are stored inline in the library JSON object in `localStorage`
- PWA folder handles (File System Access API) are persisted in IndexedDB (`folio_pwa` store) so the browser can re-open the library folder across sessions without re-prompting

**Two operating modes**
- **Standard mode** — files are uploaded individually via the Add Book modal and stored as blob URLs or base64 data in localStorage
- **PWA folder mode** — available when the File System Access API is supported; the user grants access to a root folder once and Folio scans all subfolders automatically on each launch

### Library

**Book cards**
Each book is displayed as a card in a responsive grid. Cards show:
- An emoji cover icon (🎧 for audiobooks, 📖 for ebook-only books)
- Book title (two-line clamp)
- Reading progress as a percentage and a thin progress bar at the bottom of the cover area
- A "Not started" label for unread books

**Add Book modal**
Books are added through a modal with three file slots:
- Audio file (MP3, M4A, M4B, OGG, WAV, AAC, FLAC, OPUS)
- Ebook (TXT, EPUB, HTML, HTM, MD, XHTML)
- Transcript — optional (JSON or TXT)

Each slot supports click-to-browse, drag and drop, and a clear button. A folder browse option (`webkitdirectory`) lets users select an entire book folder and auto-assigns files to the correct slots by extension. A file list is shown below with manual reassignment buttons for edge cases.

The title field auto-populates from the first assigned filename and can be edited before saving.

**Cover image support** *(added during v1.0 development)*
A fourth file slot for cover images (JPG, JPEG, PNG, WEBP, GIF, AVIF) was added to the Add Book modal. In PWA folder mode, cover images are detected automatically — files named "cover" are preferred, otherwise the first image in the folder is used. Cover images are stored as data URLs.

**Transcript auto-selection** *(added during v1.0 development)*
When browsing a folder, JSON files with "transcript" in the filename are preferred over other JSON files for the transcript slot.

**Rename and delete**
Each book card has Rename (✏️) and Delete (🗑️) action buttons. Renaming uses a browser prompt. Deleting revokes any blob URLs and removes the entry from storage.

### Player

**Layout**
The player is a full-screen view with:
- A top bar containing a back button, book title, and action icons (transcript, sync, book info, options)
- A collapsible table of contents sidebar (260px, animated)
- A scrollable ebook reading area
- A bottom bar with playback controls

**Playback controls**
- Play / pause button
- Skip back / forward (±15 seconds for audio, ±3 sentences for TTS)
- Sentence nudge buttons: −10, −5, −1, +1, +5, +10
- Seek bar (audio mode only) with current time and duration display
- Playback speed: 0.5× to 3.0× in 0.1× steps, with a direct input field
- Volume slider and mute toggle (duplicated in options panel)
- Auto-scroll toggle

**Sentence navigation**
Tapping any sentence in the ebook jumps to that position. In audio+transcript mode, tapping a sentence also seeks the audio to the matching timestamp. Sentence position and audio time are saved to storage on each advance.

### Ebook Reader

**Supported formats**
| Format | Parser |
|--------|--------|
| TXT | Line-based paragraph splitter |
| Markdown (.md) | Inline Markdown stripped, paragraph-aware |
| HTML / HTM / XHTML | DOM-based text extraction |
| EPUB | JSZip (lazy-loaded from CDN); OPF spine order respected |

**Sentence splitting**
A custom sentence splitter handles standard punctuation (`.`, `!`, `?`) with heuristics for abbreviations, decimal numbers, ellipses, and quoted speech to avoid false splits. Output is an array of sentence strings used for highlighting and TTS.

**Word highlighting**
Each sentence is tokenised into individual `<span class="word">` elements at render time. The currently playing word is highlighted with a blue background using the `word-active` class. Word highlighting can be toggled on or off in the options panel and the preference is saved.

**Sentence highlighting**
The active sentence receives a highlighted border and background. On advance, the reader auto-scrolls to keep the active sentence centred using `scrollIntoView({ behavior: 'smooth', block: 'center' })`.

**Table of contents**
Chapter headings detected during ebook parsing are listed in a collapsible sidebar. Tapping a chapter entry jumps to that sentence. The active chapter is highlighted with a left border accent.

### Transcript Sync

**Whisper JSON (word-level)**
Accepts output from OpenAI Whisper with `--word_timestamps True`. The parser reads `segments[].words[]` arrays to extract per-word timestamps. Each word is matched to a sentence and sentence to a timing range. During playback the audio `timeupdate` event drives both sentence and word highlighting in real time.

**Whisper JSON (segment-level fallback)**
For Whisper output generated without `--word_timestamps`, the parser detects that `words` arrays are absent and falls back to segment-level timing. Each segment's text is split into pseudo-words with evenly distributed timestamps across the segment duration. The transcript banner indicates "Segment-level timestamps (no word timing)" in this case.

**Plain text transcript (.txt)**
A fuzzy string matching algorithm (`similarity()`) aligns each ebook sentence against lines in the transcript file. Matched sentences receive estimated timestamps based on their relative position in the audio. The banner indicates "Plain text transcript loaded".

**Audio/text anchor sync**
For cases where the audio and ebook do not start at the same point (e.g. audio includes a preface not in the ebook), users can set an audio anchor (current audio position) and a text anchor (current sentence) to define the zero point for synchronisation. Anchors are saved per book and persist across sessions.

**Transcript management**
- Transcript can be added or replaced at any time via the Transcript modal (📝 button in top bar)
- Existing transcript can be removed
- In the Book Files modal, the transcript slot shows the current filename and allows reassignment

### Text-to-Speech (TTS) Mode

When a book has an ebook but no audio file, TTS mode is used automatically.

**Voice selection**
All voices provided by the browser's `speechSynthesis` API are listed in a dropdown in the TTS bar. The list populates when voices load (some browsers load them asynchronously).

**Speed control**
A separate TTS rate slider (0.5× to 2.0×) controls `SpeechSynthesisUtterance.rate`. The value is saved per session.

**Sentence-by-sentence playback**
Each sentence is spoken as a separate utterance. On completion, the next sentence advances automatically after the configured inter-sentence pause (0–2000ms, default 500ms). The `onend` event drives advancement.

**WPM display**
The estimated reading speed in words per minute is calculated from the sentence length and the configured rate. A label in the TTS bar shows the current WPM for reference.

### PWA & Offline Support

**Installation**
`manifest.json` declares the app as a PWA with standalone display mode, orientation set to `any`, and an SVG icon. The app can be installed to the home screen on iOS, Android, and desktop Chrome/Edge.

**Service worker**
`sw.js` implements a cache-first strategy for:
- The app HTML file
- Google Fonts (Lora, DM Sans)
- JSZip (CDN)

The service worker version string (`folio-v2`) is bumped to force cache refresh on update. Blob URLs and data URLs are passed through without caching.

**PWA folder mode**
On supported browsers (Chrome, Edge, Opera — not Safari or Firefox as of release), the user selects a root folder once via `showDirectoryPicker()`. The folder handle is stored in IndexedDB. On subsequent launches, Folio requests re-verification of the stored handle (a browser security requirement) via a single-tap regrant screen.

Each subfolder is scanned for:
- One audio file (first match by extension priority)
- One ebook file (first match by extension priority)
- One transcript file (JSON preferred over TXT; files named "transcript" preferred over others)
- One cover image (files named "cover" preferred over others)

Books with neither audio nor ebook are skipped. Scanning results are sorted alphabetically by folder name.

**Progress persistence (PWA mode)**
In PWA mode, reading progress (current sentence, current word, audio time, playback rate, WPM, anchors) is stored in `localStorage` under a per-book key derived from the folder name. This persists independently of the file handles so progress survives re-grants.

### Book Info & File Management

**Book Files modal**
Accessible from the top bar, this modal shows the currently assigned files for a book:
- Audio file name and load status
- Ebook file name and load status
- Transcript file name and type
- Cover image name and status

In standard mode, clicking any slot opens a file picker to reassign that file. In PWA folder mode, audio and ebook slots are read-only (from folder) but transcript and cover can be changed manually.

**Audio re-link modal**
A separate modal for replacing the audio file of a book that is already open. The existing audio time is preserved and the new file is seeked to match on load.

**Stale handle recovery**
In PWA mode, if a file handle becomes stale (e.g. the folder was moved or renamed), a Re-link Audio File sheet appears prompting the user to locate the file manually. Position and sync settings are preserved.

### Technical Notes

- No build step or bundler required
- No npm dependencies at runtime
- JSZip loaded lazily from CDN only when an EPUB is opened
- Google Fonts loaded from CDN; offline fallback is system serif/sans
- All blob URLs for audio are revoked on book close and library deletion to prevent memory leaks
- XSS protection via `xh()` escaping function applied to all user-supplied strings rendered into HTML
- `uid()` generates unique book IDs from `Date.now()` + random suffix
- The app detects PWA standalone mode via `window.matchMedia('(display-mode: standalone)')` and `window.navigator.standalone`
- File System Access API availability is checked via `'showDirectoryPicker' in window`

---

### Known Limitations at v1.0

- All book content (ebook text, transcripts, cover images) stored in `localStorage` alongside metadata — subject to the browser's 5MB storage quota, which can be exceeded with large books or high-resolution cover images
- Transcript timing matching runs synchronously on the main thread — can cause a brief UI pause on very large Whisper files
- EPUB parsing blocks the main thread while the ZIP is being decompressed and parsed — noticeable on large files
- TTS mode does not support word-level highlighting (no `onboundary` event handling in v1.0)
- TTS skip buttons (⏮ / ⏭) are non-functional in v1.0
- Tapping a sentence while TTS is active does not restart speech from the new position
- Reading progress is not restored when refreshing the page — returns to library screen
- No keyboard shortcuts
- No lock screen / Bluetooth media controls
- Screen can sleep during audio playback on mobile

*All of the above are addressed in v1.1.*
